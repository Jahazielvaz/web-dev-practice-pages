BEST PRACTICES

Here, we're going to be going over, best practices, or in other words, the best habits we can have to maintain a clean, and simple layout

BEST PRACTICES


  • While tags are not case sensitive, keep them case sensitive, so your code can be read easily.
  • Open and close tags, should always match.
  • Use indentation effectively so that other programers can see clusters of code and compartmentalize.
  • Use consisten quotation strings.
  • Use the commments to create sections, for things to be easier to be found.
  • Distinguishing ids solely by case sensitivity is really bad practice
  • Be consistent with your quotes.


GENERAL KNOWLEDGE

  1. White spaces in html don't count, so it's up to you to find your organizational style using white spaces.
  2. Elements are not case sensitive, but CSS declarations, class, ids, and over all identifiers are.
  3. String are just a series of characters: These could include text, numbers, spaces, symbols, etc.
  4. Strings usually go in quotation marks, but for example when you're using CSS, html will usually include them for you. These next 2 methods are valid: <p id=myName>, <p id="myName">, and <p id='myName'>, are all equivalent.
  5. In html double quotes, and single quotes(also known as apostrophes) are almost interchangeable, but they have to match.
  6. Once you assing the double of the single quote, html won't close it, until it sees the same style of quote. This is useful because you can include quotation marks as part of your text
